# APPROVAL WORKFLOW

## Low Risk Change
Required:
[ ] Developer self-check
[ ] Test pass

## Medium Risk Change
Required:
[ ] Developer check
[ ] Tester check
[ ] Documentation updated

## High Risk Change
Required:
[ ] Developer approval
[ ] Tester approval
[ ] Owner approval
[ ] Rollback plan
[ ] Monitoring plan

## Critical Change
Required:
[ ] Developer approval
[ ] Tester approval
[ ] Owner approval
[ ] Security approval
[ ] Legal/business approval
[ ] Backup completed
[ ] Rollback tested
[ ] Deployment window approved
