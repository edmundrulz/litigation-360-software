# INCIDENT RESPONSE PLAN

Use this when something breaks.

## Severity Levels

SEV-1 Critical:
System down, data loss, login broken, major legal deadline risk.

SEV-2 High:
Major feature broken, document generation broken, court workflow affected.

SEV-3 Medium:
Non-critical feature broken.

SEV-4 Low:
Cosmetic or minor issue.

## Immediate Action

1. Stop further deployment
2. Record incident
3. Check logs
4. Identify affected module
5. Decide rollback or hotfix
6. Notify stakeholders
7. Apply fix
8. Retest
9. Document root cause
10. Add prevention control
