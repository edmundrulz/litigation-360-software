## Compatibility Items To Review

1. INTEGER PRIMARY KEY AUTOINCREMENT must be converted to PostgreSQL SERIAL or GENERATED BY DEFAULT AS IDENTITY.
2. TEXT fields should remain TEXT unless length limits are required.
3. DATETIME defaults should be converted to TIMESTAMP or TIMESTAMPTZ.
4. JSON stored as TEXT should be upgraded to JSONB.
5. SQLite flexible typing must be replaced with strict PostgreSQL column types.
6. Foreign keys must be explicitly enforced.
7. Indexes must be recreated manually in PostgreSQL.
8. Audit logs should be immutable and append-only.
9. Soft delete columns should be standardized.
10. Role and permission seed data should be migrated first.

## Recommended PostgreSQL Strategy
Use environment-based database adapter:

- SQLite for local development fallback
- PostgreSQL for production
- Same service layer where possible
- Separate migration scripts
- No direct overwrite of SQLite
